# Bullet Point Guidelines (STAR/XYZ/CAR)

This is the gospel for writing resume bullets. Every bullet MUST follow one of these frameworks.

## The Three Frameworks

### STAR: Situation, Task, Action, Result

Structure: [Context of situation] → [Your specific task] → [Action you took] → [Measurable result]

**Example:**
- Reduced API response times by 40% by identifying N+1 query issues in the order processing system and implementing eager loading with Redis caching

### XYZ: Accomplished [X] as measured by [Y], by doing [Z]

This is Google's preferred format.

**Example:**
- Increased user retention by 25% (as measured by 30-day active users) by redesigning the onboarding flow and implementing personalized recommendations

### CAR: Challenge, Action, Result

Structure: [Problem/Challenge] → [What you did] → [Outcome with metrics]

**Example:**
- Resolved critical payment processing failures affecting 10K daily transactions by implementing idempotency keys and retry logic with exponential backoff

## Questions to Generate Content

### What did I do?
- Did I solve a problem?
- Did I work independently or on a team? How many people?
- What was the scope/scale?

### How did I do it?
- Why did I choose this approach?
- What tools/technologies did I use?
- Did I do testing/validation?

### What were the results?
- On time? Under budget?
- Did it work as intended?
- What metrics improved?
- What was the business impact?

## Quantification Examples

**Bad:** Improved system performance
**Good:** Reduced page load time from 4.2s to 1.1s, improving conversion rate by 18%

**Bad:** Built a data pipeline
**Good:** Built ETL pipeline processing 2M daily records with 99.9% uptime, reducing data latency from 4 hours to 15 minutes

**Bad:** Worked on authentication system
**Good:** Architected OAuth 2.0 authentication system handling 50K daily logins with <100ms response time

## Metrics to Consider

- Time saved (hours/week, percentage reduction)
- Money saved/generated
- Users/customers impacted
- Performance improvements (latency, throughput)
- Error rate reductions
- Scale (records processed, requests handled)
- Team size led/mentored
- Uptime/reliability percentages

## Integration Matters

Don't just list components. Explain HOW you integrated them to achieve the goal.

**Bad:** Used React, Redux, and PostgreSQL
**Good:** Architected real-time dashboard using React with Redux for state management, processing 10K WebSocket events/second from PostgreSQL logical replication

## Action Verb Reference

### Strong Verbs (USE THESE)
- Technical: Architected, Automated, Built, Debugged, Deployed, Designed, Developed, Engineered, Implemented, Integrated, Migrated, Optimized, Refactored, Scaled
- Impact: Achieved, Accelerated, Decreased, Eliminated, Improved, Increased, Reduced, Streamlined
- Leadership: Directed, Established, Led, Mentored, Spearheaded

### Weak Verbs (NEVER USE)
- Vague: Assisted, Helped, Participated, Worked on
- Passive: Was responsible for, Was involved in
- Redundant: Utilized (use "used"), Leveraged
- Fluff: Enhanced, Innovated, Pioneered, Revolutionized

## Common Mistakes

1. **No metrics**: "Improved performance" → Add numbers!
2. **Duty description**: "Responsible for testing" → What did you achieve?
3. **Tech list**: "Used Python, AWS, Docker" → How did you use them together?
4. **Passive voice**: "System was built" → "Built system that..."
5. **First person**: "I developed..." → "Developed..."
6. **Ending with period**: Bullets don't need periods
7. **Too long**: Keep to 1-2 lines max
