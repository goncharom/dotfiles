# Resume Formatting Rules

## Document Format

- One page maximum (unless 10+ years experience)
- Standard fonts: Arial, Calibri, or Garamond
- Minimum 10.5pt font size
- Black font only (no grey)
- Minimum 0.4 inch margins
- Single-column layout
- No icons, images, or graphics
- Left-align text (never justify)
- No excessive bolding, italics, or ALL CAPS

## Date Formatting

- Use "Present" for current positions (not "Current" or "Ongoing")
- Full years only: "2023" not "'23"
- If using months: "Mar 2022" not "3/2022" or "March 2022"
- Month abbreviations: Jan, Feb, Mar, Apr, May, June, July, Aug, Sept, Oct, Nov, Dec
- Use en dashes with spaces: "Mar 2022 – Present" not "Mar 2022-Present"
- Right-align dates
- Don't include start dates for education, only graduation

## Contact Information

### Include
- Full name (top of resume)
- Email address (Gmail or Outlook, not Yahoo/AOL/Hotmail)
- GitHub profile URL (only if active): github.com/username
- Portfolio URL (only if current): portfolio.com
- City, ST (only if local to job location)

### Never Include
- Physical address or ZIP code
- Phone number (unless specifically requested)
- LinkedIn (they can search your name)
- "Email:" or "Phone:" prefixes
- https://www. in URLs
- Country code (+1)

### Format
- Plain text, no underlining or special formatting
- Separate items with pipes: email@gmail.com | github.com/user | City, ST

## Section Headers

- Use simple names: "Work Experience" not "Professional Experience"
- "Skills" not "Technical Skills"
- "Projects" not "Personal Projects" or "Academic Projects"
- No symbols in headers

## Work Experience Format

```
**[Company], [Location] | [Job Title] | [Mon YYYY – Mon YYYY]**
- [Bullet 1 - most impressive/relevant]
- [Bullet 2]
- [Bullet 3]
```

- Reverse chronological order
- Mark internships clearly: "Software Engineer Intern"
- Mark contract positions: "Software Engineer (Contract)"
- 3-5 bullets per position
- Bullets ordered by relevance/impressiveness

## Education Format

```
**[Degree]** | [University Name] | [Graduation Year]
GPA: X.XX (only if > 3.50)
```

- Only graduation year, no date range
- "Expected May 2025" for current students
- Include GPA only if > 3.50 (to 2 decimal places)
- Remove GPA once you have full-time experience
- Don't include high school
- Don't include schools where you didn't receive a degree

## Skills Format

```
Languages: Python, Java, TypeScript, Go
Frameworks: React, Django, Spring Boot
Tools: Docker, Kubernetes, AWS, PostgreSQL
```

- 3 lines maximum
- Comma-separated (no pipes or slashes)
- Proper capitalization
- Order by importance to role
- "C, C++" not "C/C++"
- "Git" not "GitHub"

## Projects Format

```
**[Project Name]** | github.com/user/project
- [Bullet with STAR/XYZ/CAR]
- [Bullet with STAR/XYZ/CAR]
```

- Don't use "Project" in the name
- Include working links only
- Capitalize titles properly
- Order by relevance to job

## Spacing and Layout

- Consistent spacing between sections
- Consistent spacing between entries
- Don't cram content
- Bullets shouldn't extend past date alignment
- No orphaned words (1-4 words on their own line)
- No hyphenated word wraps

## Grammar

- No personal pronouns (I, we, my, our, their)
- No periods at end of bullets
- Spell out abbreviations on first use
- Use digits for numbers: "8" not "eight"
- Past tense for previous roles, present for current
- Consistent formatting throughout
